# 催促くん 要件定義書

**バージョン：** 1.0  
**作成日：** 2026-02-26  
**対象フェーズ：** MVP（最小構成）

---

## 1. サービス概要

### 1.1 サービス名
催促くん

### 1.2 コアコンセプト
「お金の催促」という心理的ストレスの高い行為を，ツールのせいにして笑いに変える，**超軽量Webツール**．  
金銭管理アプリではなく，**コミュニケーションの潤滑油**として機能する．

### 1.3 解決する課題
- 催促する側が「直接言いにくい」と感じる強烈な心理的ストレスを軽減する
- 「直接言うのは角が立つから，あえてこのツールを使ったよ」という言い訳スタンスを提供する

### 1.4 ターゲットユーザー
友人・知人・同僚に対してお金の催促をしたいが，角が立つのを避けたい人

### 1.5 MVP スコープ
- **対象：** お金の催促のみ（返信催促・出欠催促は将来対応）
- **アウトプット：** コピペ用テキスト生成のみ（Web明細URL発行は将来対応）

---

## 2. 機能要件

### 2.1 入力フォーム

ユーザーが以下の4項目を入力する．

#### 2.1.1 相手のお名前（必須）
| 項目 | 内容 |
|------|------|
| 入力形式 | テキスト自由入力 |
| 変数名 | `target_name` |
| 文字数制限 | 最大30文字 |
| 例 | 田中，佐藤さん |
| バリデーション | 空欄不可 |

#### 2.1.2 件名（必須）
| 項目 | 内容 |
|------|------|
| 入力形式 | プルダウン選択 ＋ 自由テキスト入力（「その他」選択時に表示） |
| 変数名 | `subject` |
| バリデーション | 空欄不可 |

**選択肢一覧（グループ分け）**

| グループ | 選択肢 |
|----------|--------|
| 飲食 | 飲み会代，食事代，ランチ代，ディナー代，二次会代，カフェ代 |
| 娯楽・イベント | チケット代，映画代，カラオケ代，スポーツ観戦代，ライブ・コンサート代，ゲーム代，アミューズメント代 |
| 旅行・移動 | 旅行代，宿泊費，交通費，タクシー代，ガソリン代，高速代 |
| その他 | プレゼント代，買い物代，日用品代，会費，立替金，その他（自由入力） |

**自由入力の仕様**
- プルダウンで「その他（自由入力）」を選択したとき，テキスト入力フィールドを表示する
- 最大30文字

#### 2.1.3 金額（必須）
| 項目 | 内容 |
|------|------|
| 入力形式 | 数値入力（整数） |
| 変数名 | `amount` |
| 最小値 | 1 |
| 最大値 | 9,999,999 |
| 表示形式 | 入力欄の左に「¥」を固定表示 |
| バリデーション | 空欄不可，0以下不可 |

#### 2.1.4 支払期日（必須）
| 項目 | 内容 |
|------|------|
| 入力形式 | カレンダー（date picker） |
| 変数名 | `due_date` |
| 出力形式 | 「〇月〇日まで」という日本語文字列に変換してテンプレートに埋め込む |
| バリデーション | 空欄不可 |

### 2.2 テンプレート生成

#### 2.2.1 処理方式
- LLM等のAI処理は一切使用しない
- 変数（`target_name` / `subject` / `amount` / `due_date`）を，あらかじめ用意した固定テンプレート文字列に埋め込む方式

#### 2.2.2 生成するテンプレート数
3種類．それぞれトーンが異なる．

#### 2.2.3 各テンプレートの仕様

**テンプレート①：丁寧系**
- トーン：礼儀正しく，お願いベースの文体
- イメージ：目上の人や少し距離のある相手にも使えるレベル

```
{target_name}さん，先日の{subject}の件ですが，
¥{amount}のお振込みを{due_date}までにお願いできますでしょうか．
よろしくお願いいたします．

※本通知は催促くん（https://...）により自動生成されています
```

**テンプレート②：事務的系**
- トーン：感情を排した業務通知口調
- イメージ：決済サービスやシステムの自動通知メール風

```
【自動通知】支払期日のご案内

対象者：{target_name} 様
件　名：{subject}
金　額：¥{amount}
期　日：{due_date}

お手数ですが期日までのお手続きをお願いいたします．

※本通知は催促くん（https://...）により自動生成されています
```

**テンプレート③：圧力系**
- トーン：法的通知書・督促状風の硬い文体
- イメージ：笑いのネタになるレベルのやりすぎ感を意図的に演出

```
支払督促通知書

受取人　{target_name} 殿

上記の者に対し，{subject}として立替済みの
金¥{amount}也を{due_date}までに支払うよう通知する．

※本通知は催促くん（https://...）により自動生成されています
```

#### 2.2.4 ネタバラシ文言（全テンプレート共通）
全テンプレートの末尾に必ず以下を付記する．

```
※本通知は催促くん（https://...）により自動生成されています
```

**目的：** 受け取った側が「なんだ，ツール使ったのか」と笑いで解消できるようにするための安全弁．

---

### 2.3 コピー機能

#### 2.3.1 基本仕様
- 3種類のテンプレートはそれぞれ独立したコピーボタンを持つ
- 1つのボタンで対応するテンプレートのテキスト全文をクリップボードにコピーする

#### 2.3.2 スマホ対応（最重要）
- ボタンの押下面積を十分に確保する（最低44px高さ，横幅100%）
- `navigator.clipboard.writeText()` APIを使用する
- コピー成功時，ボタンのラベルを「✔ コピーしました」に変化させる（視覚的フィードバック）
- フィードバック表示は約2秒後に元のラベルに戻す
- フォールバック：`navigator.clipboard` が使えない環境では `execCommand('copy')` を使用する

## 3. 画面フロー

```
[入力フォーム画面]
  ↓ 「通知文を生成する」ボタンを押す
  ↓ バリデーション（未入力チェック）
  ↓ エラーがあれば入力フォームにエラー表示して停止
[生成結果画面]（同一ページ内でフォームを非表示にしてリザルトを表示）
  ├── テンプレート① 丁寧系     ＋ コピーボタン
  ├── テンプレート② 事務的系   ＋ コピーボタン
  └── テンプレート③ 圧力系     ＋ コピーボタン
  └── 「最初からやり直す」ボタン
        ↓ 押すとフォームに戻りリセット
```

## 4. UI・デザイン要件

### 4.1 世界観・トーン
- **「市役所のHP」「古い銀行サイト」レベルのお堅いデザイン** を意図的に採用する
- ダサさは狙い通りであり，バグではない
- 視覚的な「やりすぎ感」がネタとして機能することを目指す

### 4.2 フォント
- 和文：Noto Serif JP（見出し），Noto Sans JP（本文）
- 等幅フォント・明朝体を積極活用してお役所感を演出する

### 4.3 カラー
| 用途 | 色 |
|------|----|
| アクセント（タイトル・ボタン等） | ダークレッド系（#8b0000） |
| 背景 | オフホワイト（#f5f5f0） |
| フォーム・カード背景 | 白（#ffffff） |
| テキスト | ほぼ黒（#1a1a18） |

### 4.4 レイアウト
- 最大幅：680px（中央寄せ）
- スマートフォンファーストで設計する
- フォームとリザルトは同一ページ内で切り替える（ページ遷移なし）

---

## 5. 非機能要件

### 5.1 技術スタック
- 単一HTMLファイル（HTML ＋ CSS ＋ JavaScript）
- バックエンド・データベース・外部API：不使用（MVPフェーズ）
- フレームワーク：不使用

### 5.2 パフォーマンス
- すべての処理はクライアントサイドで完結する
- 外部通信は発生しない

### 5.3 対応環境
- スマートフォンブラウザ（iOS Safari，Android Chrome）を最優先
- PCブラウザも動作すること

---

## 6. 将来対応（MVP対象外）

以下はMVPでは実装しないが，将来的な拡張候補として記録する．

| 機能 | 概要 |
|------|------|
| Web明細URL発行 | 入力内容をもとに専用URLを発行し，相手が開くとガチ請求書風の画面が表示される |
| お金以外の催促 | グループLINEへの返信催促・出欠催促 |
| LLM連携 | テンプレートを固定文字列ではなくAI生成に切り替えることで，より自然な文章を生成する |
| テンプレート追加 | ユーモア系・超丁寧系など，トーンのバリエーションを増やす |

## 7. 用語定義

| 用語 | 定義 |
|------|------|
| コアバリュー | このサービスが提供する最大の価値．「気まずさを回避するための言い訳ツール」 |
| ネタバラシ文言 | テンプレート末尾の「本通知は催促くんにより自動生成されています」という一文．受け取り側の怒りを笑いに転換するための安全弁 |
| 圧力系 | テンプレート③のトーン．法的通知書風の文体で，ユーモアとして機能するやりすぎ感を意図的に演出したもの |
| Web明細URL | 将来機能．ユーザーが入力した内容をもとに一意のURLを発行し，相手がアクセスすると請求書風の画面を表示する仕組み |
